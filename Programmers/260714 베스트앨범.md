# [프로그래머스 42579] 베스트앨범

> 프로그래머스 | Lv. 3(정답률 57%) | 해시, 정렬 | C++

## 문제 요약
- 장르별로 가장 많이 재생된 노래를 최대 2곡씩 선택하여 베스트앨범 구성
- 수록 기준: ①장르 총 재생수 내림차순 → ②곡 재생수 내림차순 → ③고유번호 오름차순
- 장르에 곡이 하나뿐이면 한 곡만 수록

## 풀이 접근
1. 장르별로 인덱스를 매핑 (`unordered_map`)하여 장르를 정수로 변환
2. 각 장르마다 (총 재생수, 소속 곡 인덱스 목록)을 저장
3. 장르를 총 재생수 기준 내림차순 정렬
4. 각 장르 내에서 곡들을 재생수 내림차순(동률이면 인덱스 오름차순)으로 정렬 후 최대 2곡 선택
5. 시간복잡도: O(N log N)

## 내 코드
```cpp
#include <string>
#include <vector>
#include <unordered_map>
#include <algorithm>
using namespace std;
vector<int> solution(vector<string> genres, vector<int> plays) {
    vector<int> answer;
    unordered_map<string, int> mp;
    int index = 1;
    for(int i=0; i<genres.size(); i++) {
        if(mp[genres[i]] == 0) {
            mp[genres[i]] = index;
            index++;
        }
    }
    vector<pair<int, vector<int>>> v(index, {0, {}});
    for(int i=0; i<genres.size(); i++) {
        v[mp[genres[i]]].first += plays[i];
        v[mp[genres[i]]].second.push_back(i);
    }
    sort(v.begin(), v.end(), [](auto& a, auto& b) {
        return a.first > b.first;
    });
    for(auto& [i, vec] : v) {
        if(vec.empty())
            continue;
        sort(vec.begin(), vec.end(), [plays](int a, int b) {
            if(plays[a] == plays[b])
                return a < b;
            else
                return plays[a] > plays[b];
        });
        answer.push_back(vec[0]);
        if(vec.size() >= 2)
            answer.push_back(vec[1]);
    }
    
    return answer;
}
```

## 리뷰
- 장르 매핑에 index를 1부터 시작해서 0번 슬롯이 빈 채로 남는 구조. `map<string, pair<int, vector<int>>>`로 직접 장르명을 key로 쓰면 빈 슬롯 관리가 불필요해짐
- 람다에서 `plays`를 값으로 캡처하고 있는데, 데이터가 10,000 이하라 성능 이슈는 없지만 `&plays`로 참조 캡처하면 불필요한 복사를 피할 수 있음
