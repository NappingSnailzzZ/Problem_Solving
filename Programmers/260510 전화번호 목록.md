# [프로그래머스] 전화번호 목록

> 프로그래머스 | Lv. 2(정답률 66%) | 정렬, 문자열 | C++

## 문제 요약
- 전화번호 배열에서 한 번호가 다른 번호의 접두어인 경우가 있으면 false, 없으면 true
- 배열 길이 ≤ 1,000,000, 번호 길이 ≤ 20

## 풀이 접근
1. 문자열 기준으로 사전순 정렬하면 접두어 관계가 있는 번호들이 반드시 인접하게 배치됨 (예: "12", "123", "1235" 순서)
2. 인접한 쌍만 비교하면 되므로 O(N)으로 탐색 가능
3. 앞 번호가 뒷 번호보다 짧을 때, 뒷 번호의 앞부분을 잘라 비교
4. 시간복잡도 분석: O(N * KlogN)(정렬) + O(N * K)(비교) — K는 문자열 최대 길이 20

## 내 코드
```cpp
#include <string>
#include <vector>
#include <algorithm>
using namespace std;
bool solution(vector<string> phone_book) {
    sort(phone_book.begin(), phone_book.end());
    for(int i=0; i<phone_book.size()-1; i++) {
        if(phone_book[i].length()<phone_book[i+1].length()) {
            string temp = phone_book[i+1].substr(0, phone_book[i].length());
            if(temp == phone_book[i])
                return false;
        }
    }
    return true;
}
```

## 리뷰
- 문자열 사전순 정렬에서는 같은 접두어를 가진 문자열끼리 연속 배치되므로, 모든 쌍을 비교하는 O(N²)을 인접 비교 O(N)으로 줄일 수 있음
- `substr` 대신 `compare(0, len, other)`를 쓰면 임시 문자열 생성 없이 비교 가능하여 약간의 성능 이점이 있음
