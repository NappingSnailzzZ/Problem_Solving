# [프로그래머스] 뒤에 있는 큰 수 찾기

> 프로그래머스 | Lv. 2(정답률 64%) | 스택 | C++

## 문제 요약
- 배열의 각 원소에 대해 자신보다 뒤에 있으면서 자신보다 크고 가장 가까운 수(뒷 큰수)를 구하기
- 뒷 큰수가 없으면 -1
- 배열 길이 ≤ 1,000,000

## 풀이 접근
1. 처음에 각 원소마다 앞에서부터 뒤쪽을 순회하는 브루트포스로 접근 → O(N²)으로 시간 초과
2. 스택에 아직 뒷 큰수를 못 찾은 인덱스를 쌓아두고, 새로운 원소가 스택 top보다 크면 그때 답을 확정하고 pop
3. 시간복잡도 분석: O(N), 공간복잡도 O(N)(각 원소는 스택에 최대 한 번 push, 한 번 pop되므로 전체 O(N))

## 내 코드
```cpp
#include <string>
#include <vector>
#include <stack>
using namespace std;
vector<int> solution(vector<int> numbers) {
    vector<int> answer(numbers.size(), -1);
    stack<int> st;
    
    for(int i=0; i<numbers.size(); i++) {
        while(!st.empty() && numbers[st.top()] < numbers[i]) {
            answer[st.top()] = numbers[i];
            st.pop();
        }
        st.push(i);
    }
    return answer;
}
```

## 리뷰
- 스택에 값이 아닌 인덱스를 저장해야 answer 배열의 올바른 위치에 값을 넣을 수 있음
- answer를 -1로 초기화하면 스택에 끝까지 남는 원소들은 별도 처리 없이 자동으로 -1이 됨
