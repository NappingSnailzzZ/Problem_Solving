# [프로그래머스] 숫자 카드 나누기

> 프로그래머스 | Lv. 2(정답률 55%) | GCD | C++

## 문제 요약
- 철수와 영희가 카드를 절반씩 나눠 가짐
- 한쪽 카드 전부를 나눌 수 있으면서 다른 쪽 카드는 하나도 나눌 수 없는 가장 큰 양의 정수 a를 구하기
- 배열 길이 ≤ 500,000, 원소 ≤ 100,000,000

## 풀이 접근
1. 한쪽 배열의 모든 수를 나누는 가장 큰 수 = 그 배열의 GCD
2. arrayA의 GCD를 구한 뒤 arrayB의 모든 원소를 나누는지 검증, arrayB도 마찬가지로 검증
3. 두 후보 중 큰 값을 반환하고, 둘 다 조건 불만족이면 0 반환
4. 시간복잡도 분석: O(N * log(max_val)) — GCD 계산 O(NlogM) + 검증 O(N)

## 내 코드
```cpp
#include <string>
#include <vector>
#include <numeric>
#include <algorithm>
using namespace std;
int solution(vector<int> arrayA, vector<int> arrayB) {
    int a = arrayA[0];
    int b = arrayB[0];
    
    for(int i=1; i<arrayA.size(); i++) {
        a = gcd(a, arrayA[i]);
        b = gcd(b, arrayB[i]);
    }
    
    for(int i=0; i<arrayA.size(); i++) {
        if(arrayB[i]%a == 0) {
            a = 0;
            break;
        }
    }
    
    for(int i=0; i<arrayB.size(); i++) {
        if(arrayA[i]%b == 0) {
            b = 0;
            break;
        }
    }
    
    return max(a, b);
}
```

## 리뷰
- 배열 크기 1일 때 예외 처리를 해야 한다고 생각했지만 예외 처리 없이도 정상 동작함.
- C++17의 `<numeric>` 헤더에 `std::gcd`가 있어 직접 구현 없이 바로 사용 가능
