# [Codetree] 가운데에서 시작하여 빙빙 돌기

> Codetree | Hard(정답률 74%) | 시뮬레이션, 나선 채우기 | C++

## 문제 요약
- N×N 격자의 정중앙에서 시작하여 1부터 숫자를 채움
- 이동 순서: 오른쪽 → 위 → 왼쪽 → 아래, 반복하며 회전
- N은 항상 홀수, 1 ≤ N ≤ 100

## 풀이 접근
1. 중앙 (n/2, n/2)에서 시작, 방향 순서는 →↑←↓ (d=0,1,2,3)
2. 핵심 아이디어: 나선의 "층(m)"을 관리하여 이동 가능 범위를 확장
   - m=1이면 중앙 기준 ±1 범위, m=2이면 ±2 범위
   - 현재 층의 범위 안이면 전진, 벗어나면 방향 전환
   - 현재 층이 꽉 차면 (`(2m+1)² == num`) 다음 층으로 확장
3. 방향 전환과 층 확장을 분리해서 처리한 점이 구조적으로 깔끔
4. 시간복잡도: O(N²)

## 내 코드
```cpp
#include <iostream>
#include <vector>
using namespace std;
bool inRange(int n, int m, int x, int y) {
    return (x>=n/2-m && x<=n/2+m && y>=n/2-m && y<=n/2+m);
}
int main() {
    int n, m=1;
    cin >> n;
    vector<vector<int>> v(n, vector<int>(n, 0));
    vector<int> dx = {0, -1, 0, 1};
    vector<int> dy = {1, 0, -1, 0};
    int x=n/2, y=n/2, d=0, num=1;
    v[x][y] = num;
    while(num<n*n) {
        if(inRange(n, m, x+dx[d], y+dy[d])) {
            x += dx[d];
            y += dy[d];
            num++;
            v[x][y] = num;
        }
        else if((m*2+1)*(m*2+1)==num)
            m++;
        else
            d = (d+1)%4;
    }
    for(int i=0; i<n; i++) {
        for(int j=0; j<n; j++) {
            cout << v[i][j] << ' ';
        }
        cout << endl;
    }
    return 0;
}
```

## 리뷰
- `(2m+1)² == num` 조건으로 층 확장 시점을 판단하는 게 핵심. 각 층의 마지막 숫자가 정확히 (2m+1)²이라는 수학적 성질을 활용
- while 루프 안에서 전진 / 층 확장 / 방향 전환이 한 반복에 하나만 실행되는 구조라, 무한루프 걱정 없이 안전하게 동작
- 나선 문제 변형(바깥→안, 시계/반시계 등)에서도 방향 배열 순서와 시작점만 바꾸면 동일한 프레임워크로 풀 수 있음
