# [Codetree] 코딩톡

> Codetree | Hard(정답률 26%) | 시뮬레이션, 논리 추론 | C++

## 문제 요약
- N명(A~)이 단체 대화방에서 채팅, M개의 메시지 정보가 시간순으로 주어짐
- 각 메시지마다 보낸 사람(c)과 읽지 않은 사람 수(u)가 주어짐
- 참여 중인 사람은 그 시점 이전의 모든 메시지를 읽음
- p번째 메시지를 읽지 않았을 가능성이 있는 사람들을 사전순으로 출력

## 풀이 접근
1. 메시지를 보낸 사람은 그 시점에 온라인 → 해당 시점 이전의 모든 메시지를 읽은 것이 확정
2. 따라서 p번째 이후에 메시지를 보낸 사람은 반드시 p를 읽음 → 후보에서 제외
3. 핵심 관찰: p번째와 그 직전 메시지의 읽지 않은 수(u)가 같으면, 직전 메시지의 발신자도 p를 읽은 것으로 확정 가능. 왜냐하면 직전 메시지를 보내며 온라인이었는데 u가 줄지 않았다면 p까지 계속 읽고 있었다는 뜻
4. 같은 u값을 가진 연속 구간의 시작까지 역추적 → 그 지점부터 끝까지의 모든 발신자를 제외
5. 남은 사람이 "읽지 않았을 가능성이 있는 사람"
6. 시간복잡도: O(M + N)

## 내 코드
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n, m, p, u;
    char c;
    cin >> n >> m >> p;
    vector<pair<char, int>> v(m);
    vector<int> pro(n, 0);
    for(int i=0; i<m; i++) {
        cin >> v[i].first >> v[i].second;
    }
    if(v[p-1].second==0) {
        cout << "";
        return 0;
    }
    while(p>1 && v[p-1].second==v[p-2].second)
        p--;
    for(int i=p-1; i<m; i++) {
        pro[v[i].first-'A'] = 1;
    }
    for(int i=0; i<n; i++) {
        if(pro[i] == 0) {
            c = 'A' + i;
            cout << c << ' ';
        }
    }
    return 0;
}
```

## 리뷰
- "온라인이면 이전 메시지를 전부 읽는다"는 조건이 핵심. 이 조건 덕분에 메시지 발신 = 해당 시점까지 전부 읽음이 보장됨
- "확정 읽음"과 "불확정"을 구분하는 사고방식이 필요한 문제
- 같은 u값의 연속 구간을 역추적하는 아이디어가 이 풀이의 차별점. 단순히 p 이후 발신자만 제외하면 놓치는 케이스가 생김
- u가 0인 경우 빈 문자열 출력으로 조기 리턴하는 엣지 케이스 처리도 잘 되어 있음
- 비슷한 "가능성 있는 후보 찾기" 유형에서는, 확정적으로 제외할 수 있는 조건부터 정리하고 나머지를 후보로 남기는 소거법이 효과적
