# [Codetree] 신기한 타일 뒤집기

> Codetree | Medium(정답률 52%) | 시뮬레이션, 해시맵 | C++

## 문제 요약
- 무한히 나열된 회색 타일 위에서 N번의 명령을 수행
- `x L`: 현재 위치 포함 왼쪽으로 x칸 뒤집기 → 흰색, 마지막 뒤집은 곳에 정지
- `x R`: 현재 위치 포함 오른쪽으로 x칸 뒤집기 → 검은색, 마지막 뒤집은 곳에 정지
- 기존 색과 무관하게 방향에 따라 색이 결정됨 (L=흰, R=검)
- 모든 명령 후 흰색, 검은색 타일 수 출력

## 풀이 접근
1. 타일이 무한하므로 배열 대신 `unordered_map<int, char>`로 뒤집힌 타일만 기록
2. 현재 위치를 기준으로 방향에 따라 x칸씩 순회하며 색을 덮어씀
3. 같은 타일이 여러 번 뒤집혀도 마지막 명령의 색만 남으면 되므로 단순 대입으로 처리
4. 시간복잡도: O(N × x) → 최악 O(1000 × 100) = O(100,000)

## 내 코드
```cpp
#include <iostream>
#include <unordered_map>
using namespace std;

int main() {
    int x, n;
    cin >> n;
    char direction;
    int location = 0;
    unordered_map<int, char> tiles;
    for(int i=0; i<n; i++) {
        cin >> x >> direction;
        if(direction == 'R') {
            for(int j=0; j<x; j++) {
                tiles[location+j] = 'b';
            }
            location += (x-1);
        }
        else {
            for(int j=0; j<x; j++) {
                tiles[location-j] = 'w';
            }
            location -= (x-1);
        }
    }
    int white = 0, black = 0;
    for(auto& [it, color] : tiles) {
        if(color == 'b')
            black++;
        else
            white++;
    }
    cout << white << ' ' << black;
    return 0;
}
```
