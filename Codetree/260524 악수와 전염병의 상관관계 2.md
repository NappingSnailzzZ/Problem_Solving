# [Codetree] 악수와 전염병의 상관관계 2

> Codetree | Hard(정답률 35%) | 시뮬레이션, 정렬 | C++

## 문제 요약
- N명의 개발자 중 P번이 최초 감염자
- 감염된 개발자는 정확히 K번의 악수 동안만 전염 가능, 이후엔 감염 상태이지만 전파 불가
- 감염자끼리 악수해도 전파 횟수에 포함(단, 재감염은 없음)
- T개의 악수 정보가 시간순으로 주어질 때, 최종 감염 여부를 출력

## 풀이 접근
1. 감염 상태를 3단계로 분류: 0(미감염), 1(전염 가능 감염), 2(전염 불가 감염)
2. `map<int, pair<int,int>>`으로 시간을 key로 저장 → 자동 시간순 정렬
3. 각 악수마다 분기 처리:
   - 둘 다 전파 불가 → skip
   - 한쪽만 전파 가능 → 상대 감염 + 전파자 카운트 증가
   - 둘 다 전파 가능 → 양쪽 모두 카운트 증가 (감염 상태는 변하지 않음)
   - 한쪽 전파 가능 + 한쪽 전파 불가 감염 → 전파 가능한 쪽만 카운트 증가
4. 카운트가 K에 도달하면 상태를 2로 전환
5. 시간복잡도: O(T log T) (map 삽입 정렬)

## 내 코드
```cpp
#include <iostream>
#include <map>
#include <vector>
using namespace std;

int main() {
    int n, k, p, t, a, b, c;
    cin >> n >> k >> p >> t;
    vector<int> answer(n+1, 0); // 0:미감염, 1:전염 가능 감염, 2:전염 불가능 감염
    vector<int> cnt(n+1, 0);
    answer[p] = 1;
    map<int, pair<int, int>> m;
    for(int i=0; i<t; i++) {
        cin >> a >> b >> c;
        m[a] = {b, c};
    }
    for(auto& [it, val] : m) {
        int a = val.first, b = val.second;
        if(answer[a]!=1 && answer[b]!=1)
            continue;
        else if(answer[a]==1 && answer[b]==0) {
            answer[b] = 1;
            cnt[a]++;
            if(cnt[a] == k)
                answer[a] = 2;
        }
        else if(answer[a]==0 && answer[b]==1) {
            answer[a] = 1;
            cnt[b]++;
            if(cnt[b] == k)
                answer[b] = 2;
        }
        else if(answer[a]==1 || answer[b]==1) {
            if(answer[a]==1)
                cnt[a]++;
            if(answer[b]==1)
                cnt[b]++;
            if(cnt[a] == k)
                answer[a] = 2;
            if(cnt[b] == k)
                answer[b] = 2;
        }
    }
    for(int i=1; i<answer.size(); i++) {
        if(answer[i] == 0)
            cout << 0;
        else
            cout << 1;
    }
    return 0;
}
```

## 리뷰
- 상태를 0/1/2 세 단계로 나눈 설계가 분기 처리를 깔끔하게 만들어 줌
- `map`을 사용해 시간순 정렬을 자동으로 해결한 점이 좋음. t값이 모두 다르다는 보장이 있으므로 key 충돌 걱정은 없음
