# [Codetree] 흰검 칠하기

> Codetree | Hard (정답률 43%) | 구현, 해시맵 | C++

## 문제 요약
- 무한한 일직선 타일 위에서 N번의 명령을 수행
- `x R`: 오른쪽으로 현재 포함 x칸 검은색, `x L`: 왼쪽으로 현재 포함 x칸 흰색
- 각 명령 후 마지막 칠한 타일에 위치
- 흰색·검은색 각각 2번 이상 칠해진 타일은 회색으로 고정
- 최종 흰색, 검은색, 회색 타일 수 출력

## 풀이 접근
1. 타일 좌표가 무한 범위이므로 배열 대신 unordered_map으로 방문한 타일만 관리
2. 구조체에 흰색 칠 횟수, 검은색 칠 횟수, 마지막 색을 저장
3. 각 명령마다 해당 범위 타일의 카운트 증가 및 마지막 색 갱신
4. 최종 집계: w≥2 && b≥2이면 회색, 아니면 마지막 색으로 판정
5. 시간복잡도 분석: O(N × x), 최악 O(N × 100) = O(100,000)

## 내 코드
```cpp
#include <iostream>
#include <unordered_map>
using namespace std;

struct tile {
    int w = 0;
    int b = 0;
    char color = 'b';
};

int main() {
    int n;
    cin >> n;
    int x;
    char c;
    unordered_map<int, tile> tiles;
    int location = 0;
    for(int i=0; i<n; i++) {
        cin >> x >> c;
        if(c == 'R') {
            for(int j=0; j<x; j++) {
                tiles[location+j].b++;
                tiles[location+j].color = 'b';
            }
            location += (x-1);
        }
        else {
            for(int j=0; j<x; j++) {
                tiles[location-j].w++;
                tiles[location-j].color = 'w';
            }
            location -= (x-1);
        }
    }
    int white = 0, gray = 0, black = 0;
    for(auto& [i, it] : tiles) {
        if(it.b>=2 && it.w>=2)
            gray++;
        else if(it.color == 'b')
            black++;
        else
            white++;
    }
    cout << white << ' ' << black << ' ' << gray;
    return 0;
}
```

## 리뷰
- 좌표가 무한 범위라 해시맵이 적절함
- 구조체로 타일 상태를 묶어서 관리한 점이 깔끔함
- 문제에서 "회색이 되면 더 이상 바뀌지 않는다"고 했는데, 코드는 이미 회색인 타일에도 계속 카운트를 증가시키고 color를 덮어씀. 회색 여부를 미리 확인하면 더 정확함
