# [LeetCode] 15. 3Sum

> LeetCode | Medium(Acceptance 39.0%) | 투 포인터, 정렬 | C++

## 문제 요약
- 정수 배열에서 합이 0이 되는 세 원소 조합을 모두 찾기
- 중복 불허
- 배열 길이 ≤ 3,000

## 풀이 접근
1. 배열을 정렬한 뒤, 첫 번째 원소 p를 고정하고 나머지 구간(p 이후)에서 투 포인터(l, r)로 두 수를 탐색
2. 합이 0보다 크면 r--, 작으면 l++, 같으면 결과에 추가 후 양쪽 모두 이동
3. 중복 제거: p, l, r 각각 시행
4. 시간복잡도 분석: O(N²) — 정렬 O(NlogN) + 외부 루프 O(N) × 투 포인터 O(N)

## 내 코드
```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        vector<vector<int>> answer;

        int p = 0;
        while(p < nums.size()-2) {
            int l = p+1;
            int r = nums.size()-1;
            
            while(l<r) {
                if(nums[p]+nums[l]+nums[r] > 0)
                    r--;
                else if(nums[p]+nums[l]+nums[r] < 0)
                    l++;
                else if(nums[p]+nums[l]+nums[r] == 0) {
                    vector<int> temp = {nums[p], nums[l], nums[r]};
                    answer.push_back(temp);
                    l++;
                    r--;
                    while(l<r && nums[l-1] == nums[l])
                        l++;
                    while(l<r && nums[r+1] == nums[r])
                        r--;
                }
            }
            p++;
            while(p<nums.size()-2 && nums[p-1] == nums[p])
                p++;
            }
            return answer;
        }
    };
```

## 리뷰
- 정렬 + 투 포인터 조합. O(N³) → O(N²)으로 줄이는 핵심이 투 포인터
- 중복 제거를 세 곳(p, l, r)에서 각각 처리해야 함. 하나라도 빠지면 중복이 발생
